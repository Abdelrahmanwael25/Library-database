Library Management Database System (SQL Server)
📌 Project Overview
A comprehensive relational database designed to manage library operations efficiently. This project focuses on building a robust schema to handle books, customers, employees, and the borrowing process, ensuring data integrity and optimized query performance.

🚀 Key Features
Relational Schema Design: Implementation of primary keys, foreign keys, and constraints (NOT NULL, CHECK, IDENTITY).

Advanced Data Retrieval: Developed complex SQL queries using JOINs, GROUP BY, HAVING, and Aggregation Functions.

Data Insights: * Calculated the average price of borrowed books.

Tracked customer borrowing frequency and location-based distribution.

Ranked the most borrowed books based on history.

Efficient Management: Automated identity generation and cascading deletes for data consistency.

🛠️ Tech Stack
Database Engine: Microsoft SQL Server (T-SQL)

Tools: SQL Server Management Studio (SSMS)

📁 Database Entities
Books: Stores titles, authors, page counts, and prices.

Customers: Maintains borrower personal information and contact details.

Store & Employees: Manages library branches and staff assignments.

Borrow Records: Tracks the lending process, including dates and book-customer links.

📈 Sample Queries Included
Top Borrowed Books: Identifying most popular titles.

Customer Activity: Counting borrows per user.

Branch Performance: Analyzing employee distribution per store.
